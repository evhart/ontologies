Abstract

This specification describes the CURIO language, defined as a dictionary of named properties and classes using W3C's RDF/OWL technology.

The CURIO Vocabulary is a general purpose ontology designed for managing the collaborative discussion of user generated resources in a unified and collaborative fashion. Despite being developed in the context of the WeKnowIt project, the vocabulary can be easily adapted for a particular need thanks to its flexible resource model.

CURIO can be considered as an aggregative ontology since it aggregates and connect many concept from different standard ontology in a formalised way. This vocabulary reuse most of its concept from the SIOC vocabulary but incorporate many classes and properties from other ontology such as DCTerms, GeoOWL, CommonTag and OPO. The ontology is based around the concept of Resource. A Resource is a specific class that holds a user generated content. Currently different class exists. Particularly, the Document and Event classes enables the combination of virtual events with a real one.

This document contains a detailed description of the CURIO Core Ontology and its standard modules.

Status of This Document

CURIO has been initially developed for the WeKnowIt project as a mean for connecting the different level of intelligence found within the project. However, the ontology is generic enough for being used in a wide range of project were modeling collective intelligence is necessary. CURIO is an evolving work and not considered stable at the moment. The CURIO RDF namespace is fixed and its identifier is not expected to change.

Introduction

Collaborative platforms are emerging as a new mean for managing user generated content in order to generate a collective knowledge that can be reused among communities. This generated content produced by the interaction of many user at the same time needs a common format in order to connect all this content together. The effort of creating a common understanding of an online community has been addressed by the SIOC community. However, a common model of the information shared within these communities remains informal.

The Collaborative User Resource Interaction Ontology, or CURIO, is a set of modules that provides a common understanding of resources shared between users on a common platform. The modules model different standard object such as pictures or events according to a community perspective. This perspective requires a formal representation of the generals features found in such type of community such as comments, tags, title, localisation.

The CURIO Vocabulary described in this document create common understanding of the entities manipulated by a community of users. Developers can use this ontology to express the information generated by their user in a formal way for a better exposition of their information. CURIO is not a substitute to SIOC. Instead, CURIO is a set of common practices for defining the structured information available on a social platform. CURIO sits on the top of SIOC and enable a more formal definition of the information encountered on community websites.

Example

Here is a very basic description of a Document Resource:

<curio:Document>
  <sioc:about>mage0197.jpg</sioc:about>
  <dc:title>Fallen Tree.</dc:title>
  <dc:description>The charter square roundabout has been blocked by a fallen tree.</dc:description>
  <dc:created xsi:type='xsd:dateTime'>2001-10-26T19:32:52Z</dc:created>
</curio:Document>

This example, defines a small document resource. The document is a picture. It has a title, a description and a date. A document may have more properties.

The previous example can be encoded in RDFa as well:

<tr typeof='curio:Document'>
  <td property='sioc:about' content='image0197.jpg'><img src='image0197.jpg' /></td>
  <td property='dc:title'>Fallen Tree.</td>
  <td property='dc:description'>The charter square roundabout has been blocked by a fallen tree.</td>
  <td content='2001-10-26T19:32:52Z' datatype='xsd:dateTime' property='dc:created'>26/10/2001 at 19:32</td>
</tr>

Background

CURIO is being developed within the Organisations, Information and Knowledge (OAK) Group at the University of Sheffield. This project has received funding from the EU project WeKnowIt (ICT-215453). The name "CURIO" is an acronym for "Collaborative User Resource Interaction Ontology". CURIO is also a short word that means "curiosity". The vocabulary was previously internally named "Interaction Model" within the WeKnowIt project.

The rest of this document introduces the most important classes and properties (relationship and attribute) of CURIO. It is highly recommended to read first the SIOC specification for a better understanding of CURIO.

The specific contents of the CURIO Vocabulary are detailed in the CURIO Core Vocabulary Namespace document.

The CURIO Vocabulary description

This specification introduces CURIO as a Semantic Web vocabulary or ontology. It describes the CURIO Core Vocabulary and the terms (RDF classes and properties) that constitute it, so that Semantic Web applications can use those terms in a variety of RDF-compatible document formats and applications.

Evolution and Extension of CURIO

The CURIO ontology is identified by the namespace URI "http://purl.org/net/curio/ns#".

Revisions and extensions of SIOC are conducted through edits to the namespace document, which by convention is published on the Web at the namespace URI.

CURIO Modules

CURIO modules are extensions to the CURIO Core Vocabulary. A module provides additional features to the core ontology such as specific type of resources. Currently, CURIO has two modules: Resources (curior) and Documents (curiod).

CURIO Annotations Module

This module adds annotation functionalities to the Item and Container classes. The different classes enable the collaborative description and the collaborative argumentation of the content managed by CURIO.

Classes defined within this module: AnnotationArea, AuthorTag, AutoTag, Rating, ReaderTag, ReviewableContainer, ReviewableItem, SemanticTag, Tag, TaggedContent, UserTag.

A full definition of these classes and related properties can be found in the namespace of the CURIO Annotations module at http://purl.org/net/curio/annotations#.

CURIO Resources Module

The main idea of CURIO is the concept of Resource. A resource is a piece of information generated by a user. This module describes classes that define different type of resources.

Classes defined within this module: Event.

A full definition of these classes and related properties can be found in the namespace of the CURIO Resources module at http://purl.org/net/curio/resources#.

CURIO Documents Module

Community platforms very often deal with files or such type of document resources. These resources can be grouped under the concept of Document, a specific type of Resource. For instance, a platform may deals with specific type of data such as images or audio document. In this context it is necessary to provide a suitable representation of such kind of information.

Classes defined within this module: Audio, Document, Image, Text.

A full definition of these classes and related properties can be found in the namespace of the CURIO Resources module at http://purl.org/net/curio/documents#.

CURIO and Standards

CURIO is a contribution to the Semantic Web, the specification is written using W3C's Resource Description Framework (RDF). As a consequence, CURIO adopts XML as its representation language.

Classes

Class: curio:Agent

Agent - An agent (eg. person, group, software or physical artifact).
Status: stable
subClassOf foaf:Agent

The Agent class is simply a subclass of foaf:Agent. As a consequence, it shares all the properties of a foaf:Agent.

Class: curioa:AnnotationArea

AnnotationArea - A container that contains collaborative annotations.
Status: test
Properties include: is_reviewingContainer is_reviewingItem
Used with: has_containerReview has_itemReview
subClassOf sioc:ReviewArea

The AnnotationArea class enables the annotation of an Item and Container.

Class: curiod:Audio

Audio - An audio resource as defined within the Documents Module namespace (speech, music etc...).
Status: testing
subClassOf Document
Disjoint With: curiod:Image curiod:Text

The curior:Audio class a very basic document designed for modeling audio content. This class is defined within the Documents Module namespace.

Class: curio:AuthorTag

AuthorTag - A tag asserted by the author of a resource.
Status: stable
subClassOf UserTag
Disjoint With: AutoTag ReaderTag

The AuthorTag class is a specific type of UserTag designed for identifying that the creator of this tag is also the creator of the resource.

Class: curio:AutoTag

AutoTag - A tag asserted by the software agent.
Status: stable
subClassOf UserTag
Disjoint With: AuthorTag ReaderTag

The AutoTag class is a specific type of UserTag designed for identifying that it has been asserted by an automatic system.

Class: curio:Container

Container - A resource that contains different items.
Status: test
subClassOf Container Resource
Disjoint With: Item

The Container class is a generic class designed for grouping a set of Item.

Class: curio:Document

Document - A document resource asserted by a user (typically a media file).
Status: testing
Properties include: mimeType
subClassOf Resource TaggedContent
Disjoint With: curior:Event

The Document class specifies an object uploaded by a user on a platform. Typically, a document can be considered as a file object. The uploaded object should be connected using the sioc:about property and the associated mimeType.

Class: curior:Event

Event - The representation of a real world event as defined within the Resources Module namespace.
Status: testing
Properties include: has_document has_localisation has_participation has_subEvent is_subEventOf startTime endTime
Used with: has_event is_participationOf
subClassOf LocalisedItem
Disjoint With: curior:EventResource Document

The curior:Event class specifies an event happening in the real world. Accordingly, an event is bounded by a geo localisation and a time frame.

Class: curior:EventItem

EventItem - The representation of an event resource as defined within the Resources Module namespace.
Status: testing
Properties include: has_event
Used with: has_document
subClassOf Item
Disjoint With: curior:Event UserUpdate

The curior:EventItem class specifies a resource that participates in a Event. Typically, such type of resource is a Document that describes in a different manner a particular curior:Event.

Class: curiod:Image

Image - An image document as defined within the Documents Module namespace.
Status: testing
subClassOf Document foaf:Image
Disjoint With: curiod:Audio curiod:Text

The Image class is subclassing foaf:Image. As a consequence, it shares all the properties of a foaf:Image.

Class: curio:Item

Item - An atomic resource that may be contained within an Container.
Status: test
Properties include: has_itemReview
Used with: is_reviewingItem
subClassOf Item Resource
Disjoint With: Container

The Item class is a generic class designed for managing user generated content.

Class: curio:ItemFeed

ItemFeed - A representation of the change sets of an item.
Status: testing
subClassOf Container SubscriptionList

The ItemFeed represents the change sets of an Item. It can be assimilated to an RSS feed.

Class: curio:LocalisedItem

LocalisedItem - The resource that is geolocated.
Status: testing
Properties include: has_document has_localisation has_participation has_subEvent is_subEventOf startTime endTime
Used with: has_event is_participationOf
subClassOf Resource
Disjoint With: curior:EventResource Document

The LocalisedItem class refers to a document with an associated localisation. This class may be used for assigning GPS coordinates to a picture or any other type of document.

Class: curior:Participation

Participation - An participation as defined within the Resources Module namespace.
Status: testing
Properties include: has_participant is_participationOf role
Used with: has_participation is_participatingIn

The curior:Participation class attributes link a user to an event resource through a role. A role defines how a user is involved within an event.

Class: curio:Rating

Rating - A decimal value (0..1) defining the rating of a resource.
Status: testing
Properties include: Rating
subClassOf Comment

The Rating class is a particular comment. It enables a user to assign a rating to a particular resource. The value is a decimal value between 0 and 1. The best rating is 1. The worst 0.

Class: curio:ReaderTag

ReaderTag - A tag asserted by an agent while reading a resource.
Status: stable
subClassOf UserTag
Disjoint With: AutoTag ReaderTag

The ReaderTag class is a specific type of UserTag designed for identifying that it has been asserted by an agent that has not created the tagged resource.

Class: curio:Resource

AbstractResource - An internal model of a Resource (this class should not be instantiated directly).
Status: stable
Properties include: dc:created dc:source dc:title
subClassOf foaf:Document
has subclass Container Item

The Resource class is class modeling some attribute of a Resource. This class should not be instantiated directly.

Class: curioa:ReviewableContainer

ReviewableContainer - A resource that contains different items and is annotated.
Status: test
Properties include: has_containerReview
Used with: is_reviewingContainer
subClassOf Container TaggedContent
Disjoint With: ReviewableItem

The ReviewableItem class is a generic class designed for grouping a set of Item and contains annotations.

Class: curioa:ReviewableItem

ReviewableItem - An atomic resource that may be contained within an Container and is annotated.
Status: test
Properties include: has_itemReview
Used with: is_reviewingItem
subClassOf Item TaggedContent
Disjoint With: ReviewableContainer

The ReviewableItem class is a generic class designed for managing user generated content.

Class: curioa:SemanticTag

SemanticTag - A tag with an associated meaning.
Status: testing
Properties include: means
subClassOf Tag

The SemanticTag is a normal Tag class with an additional means property. This property enables the association of tag and a resource that gives a meaning to a tag label.

Class: curioa:Tag

Tag - A free text label associated to a resource.
Status: stable
Properties include: frequency label
Used with: tag
subClassOf Tag
has subclass SemanticTag

The Tag is free text label associated to a resource. The frequency property may be used for representing the tag cloud of a resource.

Class: curioa:TaggedContent

TaggedContent - A resource that has been tagged.
Status: stable
Properties include: tagged
Used with: tagging
has subclass Resource User

The TaggedContent class identifies a resource with associated Tag.

Class: curiod:Text

Text - A textual document as defined within the Documents Module namespace.
Status: testing
Properties include: text
subClassOf Document
Disjoint With: curiod:Audio curiod:Image

The curiod:Text class defines a textual document uploaded by a user.

Class: curio:User

Text - A user contributing to the platform.
Status: stable
Properties include: is_participatingIn
Used with: has_participant
subClassOf TaggedContent sioc:User

A User class defines a person using a collaborative platform.

Class: curio:UserTag

UserTag - A tag asserted on a given resource.
Status: stable
Properties include: creator_of tag tagging taggingDate
has subclass AuthorTag AutoTag ReaderTag

A UserTag class defines a tag assertion on a resource by a user of the system.

Class: curio:UserUpdate

UserUpdate - A list of changes of a user profile.
Status: testing
subClassOf AbstractResource
Disjoint With: Document curior:Event curior:EventResource Resource

A UserTag class defines a tag assertion on a resource by a user of the system.

Properties

This documentation is work in progress. Some properties are not documented yet.

Property: curiod:text

text - The content of a textual document.
Status: testing
Domain: A thing
Range: curiod:Text

The curiod:text property specifies the content of a textual document.
